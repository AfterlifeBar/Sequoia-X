# ADR-0001：fork 定位为「上游 V2 + 个人化增强层」

- 日期：2026-07-18
- 状态：已接受

## 背景

本仓库是 sngyai/Sequoia-X 的 fork。上游在 2026-03 完成 V2 重构（分层包结构、
向量化计算、baostock 数据层），架构本身已经清晰可用；fork 侧有一系列个人自用需求
（本地备份、股池过滤、策略开关、试跑模式等）上游不会覆盖。

## 决策

fork 不重做架构，定位为「上游 V2 之上叠加个人化增强层」：

- 增强以**新增模块 + 配置项**实现（`core/universe.py`、`notify/recorder.py`、
  `core/board.py`、`.env` 配置项），对上游文件的改动尽量小。
- 范围过滤设计为**统一后置过滤**（`main.py` 中对每个策略的选股结果统一调用
  `filter_symbols()`），覆盖全部策略，而不是侵入各策略内部——策略代码保持与上游一致。
- 未来可定期从上游 rebase / merge 同步；低耦合保证同步冲突可控。

## 理由与后果

- 理由：上游架构没有需要重写的缺陷；重写会切断与上游的同步路径，得不偿失。
  后置过滤单点生效，新增策略自动获得过滤能力，也便于测试（`tests/test_universe.py`）。
- 后果：fork 与上游的 diff 需长期保持小而清晰；每次同步上游时需人工检查
  `main.py` 等少数被改动的上游文件。新增策略只需注册进 `ALL_STRATEGIES`，
  无需考虑过滤逻辑。
