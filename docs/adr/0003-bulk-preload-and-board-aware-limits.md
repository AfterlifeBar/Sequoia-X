# ADR-0003：全市场行情一次性加载 + 涨跌停阈值按板块自适配

- 日期：2026-07-18
- 状态：已接受

## 背景

两个相关的问题（PR #2）：

1. 性能：日常模式下每个策略逐股查询行情时重复建立连接 / 读库，全市场约 5200 只
   股票 × 多个策略，开销明显。
2. 正确性：旧代码用固定阈值判断"贴近涨跌停"（魔法数 1.095 / 0.905，即约 ±10%），
   但 A 股各板块涨跌停幅度不同——主板 10%、创业板/科创板 20%、北交所 30%，
   固定阈值会对创业板/科创板/北交所产生误判（假阳性/漏判）。

## 决策

1. `data/engine.py` 新增 `preload()`：日常模式在跑策略前把全市场行情一次性加载进内存，
   供逐股策略复用，避免重复开连接（`main.py` 在同步完成后调用）。
2. 新增 `core/board.py`：`limit_pct(symbol)` 按代码前缀返回板块涨跌停幅度
   （`300/301` 与 `688` → 20%，`4/8/920` → 30%，其余主板 → 10%）；
   引入容差 `LIMIT_EPS = 0.005` 判定"贴近涨跌停"，使主板行为与旧魔法数基本一致
   （(1 + 0.10) × (1 − 0.005) = 1.0945 ≈ 旧的 1.095），涨停/跌停类策略据此自适配。

## 理由与后果

- 理由：一次性加载把 I/O 从"策略数 × 股票数"降为一次全量读；按板块取阈值是
  对 A 股交易规则的正确建模，同时用容差设计保证主板既有行为不变（回归风险小）。
- 后果：内存占用增加（全市场行情常驻内存，量级对本场景可接受）；板块前缀规则
  硬编码在 `board.py`，若交易所新增板块/调整幅度需手动更新。已配
  `tests/test_board.py` 覆盖前缀映射。
